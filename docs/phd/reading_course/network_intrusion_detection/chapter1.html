Network Layers
Bits, Bytes, and Packets
  • Bit: 0 or 1
  • Bytes: 8 bits (e.g. 00101110)
      It can contain ann ASCII caracter;
      and also positive integers no greater than 255.
  • Packets: Multiple bytes grouped together for shipping across the network by packaging them.
Encapsulation
Each layer talk to its peer on other computer's stack. To do so, each leyer adds its own header (a.k.a. payload) to the data. One layer's header is another layer's data.
Interpretation of the Layers
  • Buttom line:
      Eventually we will have a bunch of 0s & 1s
  • How do we know how to interpret them?
  • Each protocol has its own layouts and formats.
Physical Addresses, MAC Addresses
  • From prespective of IP, they do not exist!
  • They are how the Ethernet card interface with the network.
  • On the contrary, the Ethernet card knows nothing about IP & IP Addresses.
  • So, a process to facilitate the correspondence is required.
ARP (Address Resolution Protocol)
  • Not an IP protocol per se.
  • For localy attached hosts only; on the same network.
  • Translate physical MAC addresses to logical IP addresses
    • The source host sends an ARP request.
    • The destination host picks it up and replies with its MAC address. (Others just ignore it.)
    • All listening hosts (including source and destination) store IP & MAC record for later use.
